# 彩色边缘标记 — 拓展说明

## 概述

本拓展将 Sobel 边缘检测输出的灰度边缘图（黑底白边或白底黑边）改为**彩色边缘**，使边缘在黑色背景上以红色、绿色或蓝色突出显示。修改范围仅限于 `hdmi_bram_sobel_display.v` 的 RGB 输出映射，不影响其他模块。

## 修改文件

| 文件 | 说明 |
| --- | --- |
| `hdmi_bram_sobel_display.v` | 新建，替代原有的黑白显示逻辑，加入彩色边缘标记 |
| `docs/彩色边缘标记_拓展说明.md` | 本文档 |

## 设计思路

### 原始逻辑

原版 `hdmi_bram_sobel_display.v` 将 `edge_mem` 中 8bit 边缘数据直接复制到 RGB 三通道：

```text
R = edge
G = edge
B = edge
```

结果为灰度边缘图（黑底白边或白底黑边）。

### 彩色边缘标记逻辑

本拓展的核心改动在 RGB 输出映射：

1. 引入 **`EDGE_THRESHOLD`** 阈值参数 — 只有当边缘强度 ≥ 阈值时，才被认为是有效边缘，显示彩色；否则显示为黑色背景。
2. 引入 **`COLOR_SEL`** 颜色选择参数 — 控制将边缘强度映射到哪个（或哪些）颜色通道。
3. 每个像素的 RGB 输出由下式决定：

```
edge_is_strong = (edge_pixel >= EDGE_THRESHOLD)

rgb_r = (de && sobel_done && edge_is_strong) ? color_r : 0
rgb_g = (de && sobel_done && edge_is_strong) ? color_g : 0
rgb_b = (de && sobel_done && edge_is_strong) ? color_b : 0

color_r = (COLOR_SEL == 0) ? edge_pixel : 0
color_g = (COLOR_SEL == 1) ? edge_pixel : 0
color_b = (COLOR_SEL == 2) ? edge_pixel : 0
```

### 效果示意

默认参数（`COLOR_SEL=0, EDGE_THRESHOLD=48`）下：
- 背景区域 → 黑色 (0, 0, 0)
- 边缘区域 → 红色，亮度与边缘强度成正比

切换为绿色（`COLOR_SEL=1`）或蓝色（`COLOR_SEL=2`）可得到对应的彩色边缘效果。

## 模块参数

### `COLOR_SEL`（颜色选择）

| 值 | 颜色 | RGB 映射 |
| --- | --- | --- |
| 0（默认） | 红色 | R = edge, G = 0, B = 0 |
| 1 | 绿色 | R = 0, G = edge, B = 0 |
| 2 | 蓝色 | R = 0, G = 0, B = edge |

### `EDGE_THRESHOLD`（边缘阈值）

- 类型：`[7:0]`，范围 0–255
- 默认值：`8'd48`
- 作用：只有 `edge_pixel ≥ EDGE_THRESHOLD` 的像素才被当作边缘并显示颜色，其余像素显示为黑色。
- 推荐取值范围：`32–80`（经验值，取决于输入图像亮度分布）

## 使用方法

### 1. 添加文件到 Vivado 工程

在 Vivado 的 Sources 面板中，右键 **Design Sources** → **Add Sources**，选择 `hdmi_bram_sobel_display.v` 并确认 **Copy sources into project**。

建议移除旧的 `hdmi_bram_sobel_display` 相关文件（如 `hdmi_bram_sobel_display - 副本.v`、`hdmi_bram_sobel_display改.v`），避免模块名冲突。

### 2. 选择颜色（修改 `hdmi_pl_top.v`）

在 `hdmi_pl_top.v` 中例化时，通过 `#()` 传递参数：

```verilog
// 红色边缘（默认，可不写参数）
hdmi_bram_sobel_display #(
    .COLOR_SEL(0),
    .EDGE_THRESHOLD(48)
) hdmi_bram_sobel_display_m0 ( ... );

// 绿色边缘
hdmi_bram_sobel_display #(
    .COLOR_SEL(1),
    .EDGE_THRESHOLD(48)
) hdmi_bram_sobel_display_m0 ( ... );

// 蓝色边缘
hdmi_bram_sobel_display #(
    .COLOR_SEL(2),
    .EDGE_THRESHOLD(48)
) hdmi_bram_sobel_display_m0 ( ... );
```

### 3. 综合、实现、生成 Bitstream

在 Vivado 中依次执行：

```
Run Synthesis
Run Implementation
Generate Bitstream
```

### 4. 下载验证

下载 bitstream 后，用串口发送图片或摄像头视频，HDMI 上应看到目标颜色的边缘图（黑色背景，彩色边缘）。

## 与其他拓展的对比

| 拓展 | 输出效果 | 核心修改 |
| --- | --- | --- |
| 固定阈值二值化 | 黑白二值边缘 | 比较 `edge_pixel ≥ TH`，输出 0 或 255 |

| **彩色边缘标记（本项）** | 黑底彩色边缘 | 阈值判断后映射到单色通道 |

## 实验建议

1. **先做基线对比**：先下载原始黑白边缘图（`edge_pixel` 直接输出到 RGB），确认 Sobel 运算和 HDMI 时序正常工作。
2. **从默认红色开始**：`COLOR_SEL=0, EDGE_THRESHOLD=48`，观察边缘是否清晰着色。
3. **调整阈值**：如果背景噪点过多，适当增大 `EDGE_THRESHOLD`（如 64、80）；如果边缘缺失过多，减小阈值（如 32、16）。
4. **尝试不同颜色**：将 `COLOR_SEL` 分别设为 0、1、2，观察同一输入在不同颜色下的视觉效果。
5. **拍照记录**：对同一张测试图，分别记录黑白边缘图和各颜色效果，附在实验报告中。
